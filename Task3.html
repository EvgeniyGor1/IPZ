Вопрос №3. Опишите отличия между командами rebase и merge в git. Когда стоит использовать rebase?

Git Merge – слияние веток. Слияние принимает содержимое ветки источника и объединяет их с целевой веткой. В этом процессе изменяется только целевая ветка. История исходных веток остается неизменной.

Git Rebase – перемещение изменения из одной ветки в другую. Rebase сжимает все изменения в один «патч». Затем он интегрирует патч в целевую ветку. В отличие от слияния, перемещение перезаписывает историю, потому что она передает завершенную работу из одной ветки в другую. В процессе устраняется нежелательная история.

Rebase лучше использовать при большом размере команды разработчиков, чтобы иметь чистую и понятную историю коммитов.
